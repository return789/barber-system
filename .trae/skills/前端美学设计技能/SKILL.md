# 前端美学设计技能

本技能指导创建独特的、生产级的前端界面，避免 generic "AI slop" 美学。实现真正可工作的代码，并对外观细节和创意选择给予非凡的关注。

## 使用场景

当用户提供以下需求时调用此技能：
- 一个组件、页面、应用程序或界面的构建需求
- 可能包含关于目的、受众或技术约束的上下文信息

## 设计思维

在编码之前，理解上下文并确定一个大胆的美学方向：

### 目的 (Purpose)
- 这个界面解决什么问题？
- 谁是它的用户？

### 调性 (Tone)
选择一个极端的方向：
- 极致极简主义 (Brutally Minimal)
- 极繁主义混乱 (Maximalist Chaos)
- 复古未来主义 (Retro-Futuristic)
- 有机/自然 (Organic/Natural)
- 奢华/精致 (Luxury/Refined)
- 玩具般趣味 (Playful/Toy-like)
- 杂志编辑风格 (Editorial/Magazine)
- 粗野主义/原始 (Brutalist/Raw)
- 艺术装饰/几何 (Art Deco/Geometric)
- 柔和/粉彩 (Soft/Pastel)
- 工业/实用 (Industrial/Utilitarian)

选择众多风格作为灵感，但设计一个忠于美学方向的方案。

### 约束 (Constraints)
- 技术需求（框架、性能、可访问性）

### 差异化 (Differentiation)
是什么让这个作品 UNFORGETTABLE（令人难忘）？人们会记住的一件事是什么？

### 关键原则
选择一个清晰的概念方向并精确执行。大胆的极繁主义和精致的极简主义都可以——关键是有意识的策划，而非强度。

## 实现要求

然后实现可工作的代码（HTML/CSS/JS、React、Vue 等），需要满足：
- 生产级且功能性完整
- 视觉上引人注目且令人难忘
- 与清晰的美学观点保持一致
- 在每个细节上都精益求精

## 前端美学指南

### 字体排版
选择美丽、独特且有趣的字体。
- 避免通用字体如 Arial 和 Inter
- 选择能提升前端美学的独特选择
- 搭配一个独特的展示字体与精致的正文字体

### 色彩与主题
- 坚持一致的美学方向
- 使用 CSS 变量保持一致性
- 主色配合锐利的点缀色优于平淡、均匀分布的调色板

### 动效
- 使用动画实现效果和微交互
- HTML 优先考虑纯 CSS 解决方案
- React 使用 Motion 库（当可用时）
- 专注于高impact时刻：一个精心编排的页面加载配合错开的揭示动画（animation-delay）比分散的微交互更能创造愉悦
- 使用滚动触发和悬停状态带来惊喜

### 空间构成
- 出人意料的布局
- 不对称、重叠、对角线流动
- 打破网格的元素
- 大量负空间 OR 控制的密度

### 背景与视觉细节
- 创造氛围和深度，而非默认纯色
- 添加与整体美学匹配的上下文效果和纹理
- 应用创意形式如：渐变网格、噪点纹理、几何图案、分层透明度、戏剧性阴影、装饰性边框、自定义光标、颗粒叠加

## 禁止使用的通用 AI 美学

- 过度使用的字体家族（Inter, Roboto, Arial, system fonts）
- 陈词滥调的配色方案（尤其是白色背景上的紫色渐变）
- 可预测的布局和组件模式
- 缺乏上下文特定个性的千篇一律的设计

## 创意执行

要创造性地解释并做出与上下文相配的意外选择。
- 不要让每个设计都相同
- 在浅色和深色主题之间变化
- 不同的字体、不同的美学
- 永远不要在不同设计中趋同于常见选择（如 Space Grotesk）

## 重要提示

实现复杂度需与美学愿景匹配：
- 极繁主义设计需要精心设计的代码，配合大量动画和效果
- 极简主义或精致设计需要克制、精确，以及对间距、字体排版和微妙细节的仔细关注

优雅来自于对愿景的良好执行。

## 输出格式

完成设计后，提供：
1. 设计决策说明（BOLD 美学方向选择）
2. 完整的、可工作的代码实现
3. 代码中包含的所有样式和动画
4. 任何必要的资源引用（如字体、图标库）
