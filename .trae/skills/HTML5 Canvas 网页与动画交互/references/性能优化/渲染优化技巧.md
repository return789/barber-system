# Canvas 渲染优化技巧

## 1. 离屏 Canvas 缓存

### 原理
将静态或半静态内容绘制到离屏 Canvas，主循环只需绘制一次离屏 Canvas，避免重复绘制复杂图形。

### 实现
```javascript
// 创建离屏 Canvas
const offscreen = document.createElement('canvas');
offscreen.width = width;
offscreen.height = height;
const offCtx = offscreen.getContext('2d');

// 预渲染复杂内容
function prerender() {
  offCtx.clearRect(0, 0, width, height);
  // 绘制复杂背景、静态元素等
  drawComplexBackground(offCtx);
  drawStaticElements(offCtx);
}

// 主循环中使用
function render() {
  ctx.clearRect(0, 0, width, height);
  ctx.drawImage(offscreen, 0, 0); // 一次性绘制
  // 绘制动态内容
  drawDynamicElements(ctx);
}
```

### 适用场景
- 复杂背景/地图
- 静态 UI 元素
- 预渲染的纹理/图案
- 不频繁变化的内容

## 2. 脏矩形渲染（Dirty Rectangles）

### 原理
只重绘发生变化的区域，而非整个画布。

### 实现
```javascript
let dirtyRects = [];

function markDirty(x, y, width, height) {
  dirtyRects.push({ x, y, width, height });
}

function render() {
  for (const rect of dirtyRects) {
    ctx.save();
    ctx.beginPath();
    ctx.rect(rect.x, rect.y, rect.width, rect.height);
    ctx.clip();
    ctx.clearRect(rect.x, rect.y, rect.width, rect.height);
    renderScene(ctx);
    ctx.restore();
  }
  dirtyRects = [];
}
```

### 适用场景
- 局部更新的场景
- UI 组件更新
- 小对象移动

## 3. 分层渲染

### 原理
使用多个 Canvas 层，静态内容和动态内容分离。

### 实现
```html
<div class="canvas-container">
  <canvas id="bg-canvas"></canvas>
  <canvas id="main-canvas"></canvas>
  <canvas id="ui-canvas"></canvas>
</div>
```

```css
.canvas-container {
  position: relative;
}
.canvas-container canvas {
  position: absolute;
  top: 0;
  left: 0;
}
```

```javascript
const bgCanvas = document.getElementById('bg-canvas');
const mainCanvas = document.getElementById('main-canvas');
const uiCanvas = document.getElementById('ui-canvas');

const bgCtx = bgCanvas.getContext('2d');    // 静态背景
const mainCtx = mainCanvas.getContext('2d'); // 动态内容
const uiCtx = uiCanvas.getContext('2d');     // UI 覆盖层
```

### 分层策略
- **背景层**：静态背景、渐变、纹理（极少更新）
- **主层**：粒子、动画、交互内容（每帧更新）
- **UI 层**：文字、按钮、指示器（按需更新）

## 4. 对象池与内存复用

### 问题
频繁创建/销毁对象导致 GC 停顿，影响帧率。

### 实现
```javascript
class ParticlePool {
  constructor(size, factoryFn) {
    this.pool = [];
    this.active = [];
    this.factoryFn = factoryFn;
    
    for (let i = 0; i < size; i++) {
      this.pool.push(factoryFn());
    }
  }
  
  acquire() {
    let obj = this.pool.pop() || this.factoryFn();
    this.active.push(obj);
    return obj;
  }
  
  release(obj) {
    const index = this.active.indexOf(obj);
    if (index > -1) {
      this.active.splice(index, 1);
      this.pool.push(obj);
    }
  }
  
  reset(obj) {
    // 重置对象状态
    obj.x = 0;
    obj.y = 0;
    obj.vx = 0;
    obj.vy = 0;
    obj.life = 0;
    obj.active = false;
  }
}

// 使用
const particlePool = new ParticlePool(10000, () => ({
  x: 0, y: 0, vx: 0, vy: 0,
  life: 0, maxLife: 0, active: false
}));

function emitParticle() {
  const p = particlePool.acquire();
  p.x = emitter.x;
  p.y = emitter.y;
  p.vx = (Math.random() - 0.5) * speed;
  p.vy = (Math.random() - 0.5) * speed;
  p.life = 0;
  p.maxLife = 1 + Math.random();
  p.active = true;
}
```

## 5. 批量绘制优化

### 问题
频繁切换样式（fillStyle, strokeStyle）导致性能下降。

### 优化策略
```javascript
// ❌ 低效：频繁切换样式
particles.forEach(p => {
  ctx.fillStyle = p.color;
  ctx.beginPath();
  ctx.arc(p.x, p.y, p.size, 0, Math.PI * 2);
  ctx.fill();
});

// ✅ 高效：按样式分组批量绘制
function renderParticles(ctx, particles) {
  // 按颜色分组
  const groups = {};
  particles.forEach(p => {
    if (!groups[p.color]) groups[p.color] = [];
    groups[p.color].push(p);
  });
  
  // 批量绘制
  for (const [color, group] of Object.entries(groups)) {
    ctx.fillStyle = color;
    ctx.beginPath();
    group.forEach(p => {
      ctx.moveTo(p.x + p.size, p.y);
      ctx.arc(p.x, p.y, p.size, 0, Math.PI * 2);
    });
    ctx.fill();
  }
}
```

## 6. 视口裁剪

### 原理
只渲染可见区域内的对象。

### 实现
```javascript
function isOnScreen(entity, viewport) {
  return entity.x + entity.radius > viewport.x &&
         entity.x - entity.radius < viewport.x + viewport.width &&
         entity.y + entity.radius > viewport.y &&
         entity.y - entity.radius < viewport.y + viewport.height;
}

function render(ctx, entities, viewport) {
  ctx.save();
  // 裁剪视口
  ctx.beginPath();
  ctx.rect(viewport.x, viewport.y, viewport.width, viewport.height);
  ctx.clip();
  
  entities.forEach(entity => {
    if (isOnScreen(entity, viewport)) {
      entity.render(ctx);
    }
  });
  
  ctx.restore();
}
```

## 7. 降采样与分辨率自适应

### 问题
高分辨率屏幕下 Canvas 像素过多，性能下降。

### 解决方案
```javascript
function setupCanvas(canvas, width, height, targetDPR = 1.5) {
  const dpr = Math.min(window.devicePixelRatio || 1, targetDPR);
  canvas.width = width * dpr;
  canvas.height = height * dpr;
  canvas.style.width = width + 'px';
  canvas.style.height = height + 'px';
  
  const ctx = canvas.getContext('2d');
  ctx.scale(dpr, dpr);
  
  return { ctx, dpr };
}

// 动态调整质量
let quality = 1.0;
function adjustQuality(fps) {
  if (fps < 30) {
    quality = Math.max(0.5, quality - 0.1);
    resizeCanvas(quality);
  } else if (fps > 55) {
    quality = Math.min(1.0, quality + 0.05);
    resizeCanvas(quality);
  }
}
```

## 9. 性能监控

### FPS 监控
```javascript
class FPSMonitor {
  constructor() {
    this.frames = 0;
    this.lastTime = performance.now();
    this.fps = 60;
  }
  
  update() {
    this.frames++;
    const now = performance.now();
    if (now - this.lastTime >= 1000) {
      this.fps = this.frames;
      this.frames = 0;
      this.lastTime = now;
    }
    return this.fps;
  }
}

// 使用
const fpsMonitor = new FPSMonitor();

function animate(time) {
  update();
  render();
  const fps = fpsMonitor.update();
  if (fps < 30) {
    console.warn('Low FPS:', fps);
  }
  requestAnimationFrame(animate);
}
```

## 10. Web Worker 辅助计算

### 原理
将密集计算移到 Worker，避免阻塞主线程。

### 实现
```javascript
// main.js
const worker = new Worker('physics-worker.js');

function updatePhysics(entities) {
  worker.postMessage({ type: 'update', entities });
}

worker.onmessage = (e) => {
  if (e.data.type === 'updated') {
    entities = e.data.entities;
  }
};

// physics-worker.js
self.onmessage = (e) => {
  if (e.data.type === 'update') {
    const entities = e.data.entities;
    // 密集物理计算
    entities.forEach(entity => {
      entity.x += entity.vx;
      entity.y += entity.vy;
      // 碰撞检测等
    });
    self.postMessage({ type: 'updated', entities });
  }
};
```

## 常见问题

### Q: Canvas 性能瓶颈在哪里？
A: 常见瓶颈：
1. 过多绘制调用（非像素数）
2. 频繁状态切换
3. GC 停顿（对象创建/销毁）
4. 未裁剪的离屏渲染
5. 主线程阻塞

### Q: 什么时候用 Canvas vs DOM？
A: 
- **Canvas**：1000+ 动态对象、像素级控制、游戏/粒子
- **DOM**：静态内容、SEO/可访问性、CSS 样式需求

### Q: 如何处理高分屏性能问题？
A: 限制 max DPR（1.5-2.0），或动态调整渲染分辨率。
