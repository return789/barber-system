# 数据库设计检查清单

## 需求分析阶段

- [ ] 完成业务需求调研
- [ ] 明确数据处理的高频操作（CRUD比例）
- [ ] 确认性能要求（SLA、响应时间）
- [ ] 确定数据安全等级
- [ ] 评估数据量级和增长趋势
- [ ] 识别核心业务实体

## 概念设计阶段

- [ ] 完成实体识别和分类
- [ ] 确定实体属性
- [ ] 绘制ER图
- [ ] 识别实体间关系（1:1、1:N、M:N）
- [ ] 验证业务场景覆盖完整性
- [ ] 与业务方确认ER图准确性

## 逻辑设计阶段

- [ ] ER图转换为关系模型
- [ ] 确定每张表的主键
- [ ] 确定外键关系
- [ ] 应用规范化（至少达到3NF）
- [ ] 处理多对多关系（创建关联表）
- [ ] 设计继承/特化关系
- [ ] 审核表结构合理性

## 物理设计阶段

### 表结构设计

- [ ] 表名符合命名规范（t_模块_实体）
- [ ] 字段名符合命名规范（下划线分隔）
- [ ] 主键统一使用id（BIGINT）
- [ ] 字段类型选择合理
- [ ] 字段长度预留合理
- [ ] 所有字段有明确注释
- [ ] 包含审计字段（created_at、updated_at、deleted_at）
- [ ] 状态字段定义状态常量
- [ ] 无NULL字段（设置默认值）
- [ ] 无预留字段（考虑扩展性设计）
- [ ] 敏感字段加密处理
- [ ] 表有表注释

### 索引设计

- [ ] 主键自动创建唯一索引
- [ ] WHERE条件字段有索引
- [ ] JOIN连接字段有索引
- [ ] ORDER BY字段有索引
- [ ] GROUP BY字段有索引
- [ ] 复合索引遵循最左前缀原则
- [ ] 索引数量控制在7个以内
- [ ] 高选择性字段优先建索引
- [ ] 索引命名规范（idx_/uk_/pk_前缀）

### 约束设计

- [ ] 主键约束
- [ ] 外键约束（考虑性能可暂时不加）
- [ ] 唯一约束
- [ ] 非空约束
- [ ] 检查约束（可选）
- [ ] 默认值设置

### 存储设计

- [ ] 选择合适的存储引擎（MySQL用InnoDB）
- [ ] 选择合适的字符集（utf8mb4）
- [ ] 选择合适的排序规则
- [ ] 评估表空间需求
- [ ] 大表考虑分区方案

## 安全性设计

- [ ] 敏感数据加密存储
- [ ] 重要操作审计日志
- [ ] 权限分级设计
- [ ] 数据备份策略

## 性能设计

- [ ] 预估查询性能
- [ ] 预估写入性能
- [ ] 考虑读写分离需求
- [ ] 考虑分库分表需求
- [ ] 缓存策略设计

## 代码评审

- [ ] SQL编写规范检查
- [ ] 索引使用合理性检查
- [ ] 事务使用正确性检查
